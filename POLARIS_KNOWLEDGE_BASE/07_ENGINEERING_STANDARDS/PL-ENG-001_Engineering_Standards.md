# PL-ENG-001 — Polaris Engineering Standards

Version: 1.0  
Status: Active

## Feature Lifecycle
Designed → Coding → Running → Verified → Committed

## Definition of Done
A feature is done only when:
- Code is written.
- Code runs.
- Feature is tested.
- GitHub is updated.
- Documentation is updated.
- Founder has verified it.

## Current Sprint
Sprint 2: Running Application

Checklist:
- React frontend runs successfully.
- FastAPI backend runs successfully.
- SQLite database is created automatically.
- Dashboard loads in a browser.
- Add Responsibility form works.
- Responsibility is saved to SQLite.
- Dashboard immediately displays the new responsibility.
- Founder verifies all of the above.
