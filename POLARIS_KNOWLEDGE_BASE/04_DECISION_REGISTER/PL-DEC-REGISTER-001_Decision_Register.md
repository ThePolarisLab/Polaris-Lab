# PL-DEC-REGISTER-001 — Polaris Decision Register

Version: 1.0  
Status: Active

## PD-001 — Create Polaris Lab
Decision: Polaris Lab becomes the institutional home for Builder-focused research, systems, software, and publications.

## PD-002 — Domain Secured
Decision: polarislab.ca becomes the initial domain identity.

## PD-003 — GitHub Becomes Institutional Memory
Decision: GitHub is the source of truth for Polaris Lab knowledge, documentation, engineering, and releases.

## PD-004 — Mor Logistics Becomes Living Laboratory
Decision: Mor Logistics Manitoba Limited is the first real-world validation environment for Polaris Lab systems.

## PD-005 — Polaris Chief of Staff Becomes First Product
Decision: The first software product will be Polaris Chief of Staff.

## PD-006 — Every Repeated Responsibility Deserves a System
Decision: Adopted as the Polaris Law.

## PD-007 — Measure Progress by Working Software
Decision: Progress will be measured by verified working software, not documents alone.

## PD-008 — Definition of Done
Decision: A feature is done only when written, running, tested, committed, documented, and Founder-verified.

## PD-009 — AI Prepares; Builder Decides
Decision: Polaris AI prepares Builders to make better decisions. It does not replace Builder judgment.

## PD-010 — GitHub Must Be Updated With Official Decisions
Decision: Important commitments, principles, roadmaps, and decisions must be moved from chat into GitHub.
