## Founder
Surinder Pahil

## Founder Principles Captured
1. System is most important.
2. No matter how small the work, there must be a system.
3. Verification builds trust.
4. Communication completes work.
5. Everyone must update the status of assigned work.
6. If work is not done, explain the reason immediately.
7. If there is an issue, ask immediately.
8. Nothing important should depend on memory when a system can remember.
9. Finish what you start.
10. Build for generations, not quarters.

## The Pahil Method
1. Accept Responsibility
2. Find the Root Cause
3. Build a System
4. Verify
5. Communicate
6. Continuously Improve
