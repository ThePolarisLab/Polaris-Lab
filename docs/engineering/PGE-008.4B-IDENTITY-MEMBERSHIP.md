# PGE-008.4B — Identity and Membership Foundation

## Status

Implementation review.

## Purpose

Establish a vendor-neutral human identity and an explicit organization membership boundary without coupling Polaris to an authentication provider.

## Capabilities

- Create and retrieve identities using stable UUIDs.
- Normalize and uniquely constrain email addresses.
- Create one explicit membership per identity and organization.
- Assign an initial role: owner, admin, member, or viewer.
- List memberships within an organization boundary.
- Publish canonical identity and membership events.

## Canonical events

- `identity.identity.created.v1`
- `identity.membership.created.v1`

Membership events carry matching `organization_id` and `tenant_id` values. Identity creation remains platform-level because an identity may later belong to more than one organization.

## Constitutional alignment

- **Builder Freedom:** identities are independent of Microsoft, Google, local-password, or future authentication providers.
- **Human Authority:** creating an identity does not grant access; membership must be explicit.
- **Explainability:** membership events identify source, actor, subject, tenant, and idempotency evidence.
- **Security by Design:** organization boundaries are persisted before authentication and authorization are introduced.

## Explicit boundaries

This increment does not implement:

- login, sessions, passwords, tokens, OAuth, or OpenID Connect;
- permission evaluation or endpoint enforcement;
- invitations, approval workflows, or role changes;
- authentication-provider account linking;
- ABAC policies;
- schema migration tooling.

The initial role field is descriptive domain state. It must not be treated as an authorization decision until the policy and enforcement milestone is delivered.

## Verification

- valid identity creation returns HTTP 201;
- duplicate normalized email returns HTTP 409;
- malformed email returns HTTP 422;
- missing identity or organization returns HTTP 404;
- duplicate membership returns HTTP 409;
- membership creation publishes a tenant-aware canonical event;
- memberships are listed only through their organization-scoped endpoint;
- existing platform tests remain green.

## Next increment

PGE-008.4C should introduce provider-neutral authentication identities, policy contracts, and request-context enforcement without embedding one identity provider into the domain model.
