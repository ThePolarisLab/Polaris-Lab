# PGE-008.5D — Runtime Event Bus

## Status

Implementation review.

## Purpose

Introduce a small, strongly typed, in-process event bus so Runtime lifecycle activity can be observed without adding direct dependencies from Runtime to metrics, health, Mission Control, Executive Memory, or Athena.

## Delivered capabilities

- canonical `IEvent` identity, name, and timestamp contract;
- typed named subscriptions;
- global subscriptions for cross-cutting observers;
- synchronous and asynchronous publication;
- stable handler invocation order within a publication;
- subscriber failure isolation through an optional error handler;
- idempotent unsubscribe functions;
- duplicate handler suppression;
- event envelopes with correlation, causation, source, and version metadata;
- Runtime lifecycle publication through the bus;
- compatibility with the existing lifecycle listener API.

## Delivery semantics

The event bus is memory-backed and process-local. `publish` invokes subscribers synchronously while isolating thrown errors. Promise rejections are reported asynchronously. `publishAsync` awaits handlers sequentially, preserving subscription order for that publication.

Subscriptions are stored in sets. Registering the same handler more than once for the same event does not duplicate delivery. A subscription returns a function that may be called repeatedly to remove the handler safely.

## Runtime integration

Lifecycle events now implement `IEvent` and receive unique event IDs. Runtime publishes each lifecycle event to the injected `IEventBus`. Downstream modules can subscribe without Runtime importing or knowing about those modules.

## Explicit limits

This increment intentionally does not provide:

- durable event storage;
- distributed brokers;
- replay or event sourcing;
- delivery acknowledgements;
- cross-process ordering guarantees;
- retry or dead-letter policies;
- schema registries.

Those capabilities should be introduced only when supported by concrete product requirements.

## Verification

Tests cover named and global delivery, unsubscribe behavior, duplicate suppression, failure isolation, asynchronous ordering, and end-to-end Runtime lifecycle publication.
