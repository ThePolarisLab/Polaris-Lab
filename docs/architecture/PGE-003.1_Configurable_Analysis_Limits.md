# PGE-003.1 — Configurable Analysis Limits

## Purpose

PGE-003.1 makes the Python source-analysis size limit configurable without removing the safety boundary introduced in PGE-003.

## Configuration

Set the optional environment variable:

```text
POLARIS_CODE_ANALYSIS_MAX_BYTES=1000000
```

The default remains 1,000,000 bytes. The hard maximum is 10,000,000 bytes.

## Resolution order

1. An explicit `PythonCodeAnalyzer(max_source_bytes=...)` value.
2. `POLARIS_CODE_ANALYSIS_MAX_BYTES`.
3. The 1,000,000-byte default.

An explicit constructor value overrides the environment variable, which supports deterministic tests and controlled internal use.

## Validation

The engine rejects:

- non-integer environment values;
- zero or negative limits;
- limits above 10,000,000 bytes.

The analyzer continues to reject source that exceeds the active limit before parsing it.

## Response metadata

Successful analysis now reports:

- `analysis_limit_bytes` — the active configured limit;
- `source_bytes` — the UTF-8 size of the analyzed source.

This makes the applied policy visible to API consumers and simplifies troubleshooting.

## Security boundary

This increment does not execute analyzed code and does not add chunked analysis. Large-file chunking remains PGE-003.2, while runtime execution remains reserved for a future isolated sandbox engine.
