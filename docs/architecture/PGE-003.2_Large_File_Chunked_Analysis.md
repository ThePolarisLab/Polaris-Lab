# PGE-003.2 — Large-file Chunked Analysis

## Status

Implemented and validated through backend CI.

## Purpose

PGE-003.2 adds syntax-aware chunked analysis for Python files that exceed the configured single-pass analysis limit while preserving the existing hard safety ceiling.

## Behavior

- single-pass analysis remains unchanged;
- chunked mode must be requested explicitly;
- chunks are created only between complete top-level Python statements;
- functions and classes are never split in the middle;
- aggregate results preserve module-level structure and metrics;
- chunk metadata includes line ranges, byte sizes, and symbol counts;
- analyzed source is never executed;
- invalid syntax returns a controlled error;
- files above the hard safety ceiling remain rejected.

## API

The following endpoints support `chunked=true`:

- `GET /api/v1/code-understanding/analyze`
- `GET /api/v1/code-understanding/explain`

## Scope boundary

Cross-file symbol resolution and repository-wide dependency graphs remain part of PGE-003.3.
