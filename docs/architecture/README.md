# Polaris Architecture

This directory contains the living architecture baseline for Polaris.

## ARC-001 status

ARC-001 establishes an evidence-based description of the repository before additional major engineering capabilities are added.

## Documents

- `01_System_Overview.md` — current system boundaries and major layers
- `02_Repository_Map.md` — verified repository structure and component inventory
- `03_Engineering_Standards.md` — working engineering rules for future changes
- `Technical_Debt.md` — observed risks and improvement backlog
- `adr/ADR-001-architecture-before-major-features.md` — first architecture decision record

## Maintenance rule

Architecture documentation must distinguish between:

1. **Verified current state** — confirmed from repository code.
2. **Proposed target state** — recommendations that have not yet been implemented.

The repository remains the source of truth.
