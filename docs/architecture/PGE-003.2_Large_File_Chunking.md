# PGE-003.2 — Large-File Chunked Analysis

## Purpose

PGE-003.2 adds explicit syntax-aware analysis for Python files that are larger than the configured single-pass analysis limit.

## Behavior

Normal analysis remains strict and continues to reject files larger than `POLARIS_CODE_ANALYSIS_MAX_BYTES`.

Large files can be analyzed explicitly with chunked mode:

- `GET /api/v1/code-understanding/analyze?path=...&chunked=true`
- `GET /api/v1/code-understanding/explain?path=...&chunked=true`

The analyzer parses the module without executing it, groups complete top-level Python statements into chunks, and returns one aggregate module analysis together with chunk metadata.

## Chunk boundaries

Chunks are formed only between complete top-level statements such as imports, constants, functions, and classes. A class or function is never split in the middle. A single symbol larger than the configured chunk target is retained as one oversized chunk so its syntax remains valid.

Each chunk reports:

- index;
- starting and ending source line;
- source byte count;
- counts of imports, functions, classes, and constants.

## Safety

- Python source is never executed.
- The configured analysis limit becomes the chunk target.
- The existing 10,000,000-byte hard ceiling remains mandatory.
- Invalid Python syntax returns a controlled analysis error.
- Single-pass behavior remains backward compatible.

## Scope boundary

PGE-003.2 does not yet resolve symbols across different files. Repository-wide dependency resolution remains PGE-003.3.
