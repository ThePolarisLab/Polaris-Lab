# PGE-008.5G — Worker SDK

## Purpose

Provide a small, strongly typed authoring layer for Polaris Runtime workers without coupling business logic to Runtime internals.

## Delivered capabilities

- `defineWorker` factory for concise worker creation;
- typed payload and result handlers;
- automatic conversion of plain handler values into `WorkerResult.success`;
- preservation of explicit `WorkerResult` failures;
- worker name and version validation;
- immutable worker definitions;
- `WorkerRegistry` for registration, lookup, listing, and Runtime installation;
- reference `EchoWorker` implementation;
- automated SDK and Runtime integration tests.

## Example

```ts
const worker = defineWorker<number, number>({
  name: "double",
  version: "1.0.0",
  defaultPayload: 21,
  handler: (_context, value) => value * 2,
});

const registry = new WorkerRegistry();
registry.register(worker);
registry.install(runtime);
```

## Architecture

The SDK depends on the existing public Runtime contracts: `IWorker`, `WorkerDefinition`, `ExecutionContext`, `WorkerResult`, and `IRuntime`. Runtime does not depend on the SDK.

## Scope boundary

This increment does not add dependency injection, schema validation, remote workers, package publication, decorators, dynamic module loading, durable registration, or payload submission through `Runtime.execute`. Those capabilities require separate milestones.
