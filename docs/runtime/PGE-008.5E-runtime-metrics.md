# PGE-008.5E — Runtime Metrics & Telemetry

## Purpose

Provide low-coupling, event-driven observability for Polaris Runtime. Runtime continues to execute work and publish lifecycle events; the metrics subsystem observes those events through `IEventBus` and exposes immutable snapshots.

## Architecture

```text
Runtime -> Lifecycle events -> EventBus -> MetricsCollector -> MetricsSnapshot
```

`MetricsCollector` is the only mutable aggregation component. Consumers depend on the read-only `IMetricsProvider` contract.

## Metrics

The first increment records:

- created, validated, queued, dispatched, started, succeeded, failed, cancelled, retry-pending, timed-out, and completed totals;
- queue depth and active worker gauges;
- queue, dispatch, and execution duration distributions;
- minimum, maximum, average, P50, P95, and P99 durations;
- success and failure rates;
- completed-worker throughput per minute.

## Lifecycle mapping

| Event | Update |
| --- | --- |
| `WorkerCreated` | increment created |
| `WorkerValidated` | increment validated |
| `WorkerQueued` | increment queued and queue depth; begin queue timer |
| `WorkerDispatched` | increment dispatched; decrement queue depth; record queue duration; begin dispatch timer |
| `WorkerStarted` | increment started and active workers; record dispatch duration; begin execution timer |
| `WorkerSucceeded` | increment succeeded |
| `WorkerFailed` | increment failed |
| `WorkerCancelled` | increment cancelled |
| `WorkerRetryPending` | increment retries |
| `WorkerTimedOut` | increment timed out |
| `WorkerCompleted` | increment completed; decrement active workers; record execution duration |

Gauges clamp at zero so incomplete or externally supplied event streams cannot produce negative queue or active-worker values.

## Public API

```ts
const collector = new MetricsCollector(eventBus);
const snapshot = collector.snapshot();
collector.stop();
```

Snapshots are frozen and contain copied aggregate values. Histogram summaries are also frozen.

## Scope boundary

This increment intentionally excludes persistence, external telemetry exporters, labels/tags, distributed aggregation, alert policies, and bounded/reservoir histograms. Those belong in later observability and health milestones when requirements are concrete.

## Verification

Tests cover metric primitives, percentiles, event-to-metric mapping, durations, rates, throughput, non-negative gauges, immutable snapshots, and unsubscribe behavior.
