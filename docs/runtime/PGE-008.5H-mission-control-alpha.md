# PGE-008.5H — Mission Control Alpha

## Purpose

Mission Control Alpha provides a single read-only operational view across the Polaris Runtime foundation.

## Delivered capabilities

- canonical `IMissionControl` read contract;
- immutable operational snapshots;
- Runtime Alpha version identity;
- registered-worker inventory;
- scheduler queue depth;
- current Runtime metrics;
- current health state and active alerts;
- deterministic clock injection for testing;
- automated aggregation and immutability tests.

## Architecture

```text
Worker Registry ─┐
Scheduler ───────┼─> Mission Control ─> MissionControlSnapshot
Metrics ─────────┤
Health ──────────┘
```

Mission Control consumes existing read-oriented contracts and does not alter Runtime execution, lifecycle, scheduling, metrics, or health behavior.

## Alpha boundary

This milestone intentionally delivers an in-process read model rather than a web UI. It excludes command execution, persistence, historical dashboards, authentication, remote transport, distributed aggregation, notifications, and automatic remediation.

Those capabilities belong to later Mission Control and Hermes increments after Runtime v0.1 Alpha certification.

## Runtime Alpha readiness

With this milestone, the Runtime foundation contains:

- Runtime Kernel;
- Lifecycle Engine;
- Scheduler Engine;
- Event Bus;
- Metrics and Telemetry;
- Health Monitoring;
- Worker SDK;
- Mission Control Alpha.
