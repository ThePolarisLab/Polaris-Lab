# PGE-008.5F — Runtime Health Monitoring

## Purpose

Runtime Health Monitoring converts immutable metrics snapshots into an operational health state for Mission Control and future Athena diagnostics.

## Architecture

`Runtime -> Event Bus -> MetricsCollector -> HealthMonitor -> Mission Control`

Runtime has no dependency on Health Monitoring. `HealthMonitor` depends only on the read-only `IMetricsProvider` contract.

## States

- `healthy`
- `degraded`
- `unhealthy`
- `critical`

The overall state is the highest severity produced by any active rule.

## Initial rules

- worker failure rate
- queue depth
- retry rate
- timeout rate
- P95 execution latency
- completed-work throughput

Thresholds are configurable through `HealthThresholds`; safe defaults are supplied by `DEFAULT_HEALTH_THRESHOLDS`.

## Public API

- `status()`
- `snapshot()`
- `alerts()`
- `isHealthy()`
- `isCritical()`

Snapshots and alerts are immutable. Health is recalculated from the latest metrics on every read, allowing immediate recovery when underlying conditions improve.

## Scope boundary

This increment excludes persistence, notification delivery, historical trend analysis, distributed aggregation, automatic remediation, and external incident-management integrations.
