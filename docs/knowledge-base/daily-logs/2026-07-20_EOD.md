## Research Notes

- Complexity is an input to smell detection, not a smell by itself.
- Long methods, deep nesting, parameter overload, excessive locals, and excessive returns can be detected locally with explainable AST evidence.
- Large-class detection requires class-level aggregation rather than only function-level metrics.
- Magic-number detection benefits from exclusions for common structural constants and from reporting exact source locations.
- Duplicate code requires normalized structural comparison across functions or files.
- Dead code requires call-graph, control-flow, or reachability evidence.
- Feature envy requires cross-object access analysis and should not be inferred from shallow syntax alone.
- Explainability and bounded confidence are essential safeguards against false-positive-heavy static analysis.
