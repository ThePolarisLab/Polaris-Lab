# Polaris v0.5.0 — Executive Memory

## Summary

Polaris v0.5.0 introduces Executive Memory, giving Athena a scoped and explainable memory domain for organizational decisions and context.

## Included

- user-scoped memory records;
- deterministic retrieval ranking using relevance, recency, and importance;
- memory update and deletion lifecycle operations;
- ownership enforcement for reads, updates, and deletes;
- Athena memory adapter integration;
- defensive cloning in the in-memory repository;
- ADR-003 architecture and guardrails;
- automated Executive Memory tests in GitHub Actions.

## Verification

The Executive Memory GitHub Actions test job passed on the release pull request before merge.

## Known boundaries

- storage is currently in-memory and not durable across processes;
- lexical ranking does not yet provide semantic similarity;
- archival and audit-history behavior remain future work;
- broader Athena TypeScript type-hardening remains a separate engineering task.
